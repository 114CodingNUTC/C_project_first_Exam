## Plan: 五子棋（終端機）精簡執行版

以 Dev C++ 可編譯的純 C 專案為目標，先完成可維護核心，再實作 UI/AI/語系。所有落子來源（玩家/AI）共用同一提交與判定流程，避免規則分叉。

**Coding Principles（硬性）**
1. 代碼精簡、單一職責：函式短小、命名清楚、避免重複邏輯。
2. 禁止硬編碼：事件/狀態/錯誤/文案皆走常數與鍵值中心（如 `EV_READY`）。
3. 所有 C 檔僅放 `src/`；輸出 `game.exe`。
4. 只用 Dev C++ 可用函式庫：`stdio.h` `stdlib.h` `string.h` `ctype.h` `time.h` `conio.h` `windows.h`。

**Steps**
1. Phase A — 架構與契約（阻塞後續）
   1.1 建立模組骨架：`src/`、`include/`、核心型別與常數。
   1.2 定義單一落子 API（玩家/AI 都必經）：`try_place_stone(...)`。
   1.3 建立 `config` / `messages` / `events` 三中心（設定、文案、事件碼）。

2. Phase B — 規則核心（depends on 1）
   2.1 棋盤操作：初始化、合法檢查、提交、回滾。
   2.2 勝負檢查：四方向連五。
   2.3 回傳終局資訊：勝方 + 成線五子座標（供高亮）。
   2.4 回合控制：換手、和局、狀態轉移。

3. Phase C — UI 與輸入（depends on 1,2）
   3.1 主選單：`1v1`、`1vsAI(簡/中/難)`、棋盤 `9/15/19/自訂`。
   3.2 1v1 身份與先手：左白右黑；先手可選左先/左後/隨機。
   3.3 輸入：`a,1` / `a1` / `1a` 文字（大小寫皆可，如 `A1` / `1A`） + 方向鍵移動 + Enter 提交（可選滑鼠點選聚焦）。
   3.4 右下角單一訊息槽：操作提示與錯誤訊息共區；錯誤紅字顯示後自動回復（預設 2 秒）。
   3.5 顯示規範：當前思考方、最後落子高亮、終局成線五子高亮。
   3.6 版面與渲染：自適應排版、防變形、防閃爍（局部重繪）。
   3.7 刷新安全：一致性檢查 + 自動修復重繪，避免殘影/錯位。

4. Phase D — AI（depends on 2，可與 C 部分並行）
   4.1 簡單：隨機合法點。
   4.2 中等：防守優先 + 基礎進攻。
   4.3 困難：Minimax + alpha-beta + 候選點縮減。
   4.4 AI 僅回傳座標，最終仍走共用落子 API。

5. Phase E — 流程整合（depends on 3,4）
   5.1 標準回合：顯示 → 取行動 → 共用落子 → 判定 → 換手。
   5.2 ESC 操作：暫停、繼續、重開、回主選單、退出（退出需確認）。
   5.3 狀態機：`InGame / Paused / MainMenu / ConfirmExit`。
   5.4 事件追蹤：關鍵節點輸出事件碼序列，便於排錯。

6. Phase F — 驗證與文件（depends on 5）
   6.1 功能矩陣：模式 × 棋盤大小 × 輸入方式 × AI 難度。
   6.2 UI 穩定：防閃爍、防刷新錯誤、最後落子高亮轉移正確。
   6.3 規則正確：成線五子、勝方判定、和局判定。
   6.4 系統化檢查：反硬編碼、語系切換完整性、config 重編譯生效。
   6.5 更新 README：建置、操作、模組邊界、事件碼與設定說明。

**Relevant files**
- `README.MD` — 規格、建置與操作說明
- `include/config.h` — 可重編譯設定
- `include/messages.h`, `src/messages.c` — 文案鍵值與語言映射
- `include/events.h` — 事件/狀態/錯誤碼
- `src/main.c`, `src/game.c`, `src/board.c`, `src/rules.c`, `src/input.c`, `src/ui.c`
- `src/ai_easy.c`, `src/ai_medium.c`, `src/ai_hard.c`

**Decisions（已鎖定）**
- 必須：單一落子流程、禁止硬編碼、事件碼化、訊息集中、語言切換、config 重編譯。
- 必須：右下角訊息槽、錯誤紅字回復、最後落子高亮、終局成線高亮。
- 必須：左白右黑、先手可設定/隨機、顯示當前思考方。
- 必須：防閃爍 + 刷新錯誤自動修復。
- 排除：GUI、網路對戰、存檔回放（本階段不做）。

**Further Considerations（第二階段）**
1. 版本資訊頁（版本/建置時間/語言/config 摘要）
2. 回合快照（未來悔棋/回放）
3. 日誌等級（ERROR/INFO/DEBUG）
4. 最小 smoke test 劇本固定化
